Static build system

The site is compiled from modular content sources. Wiki articles and blog posts live under the content directory, while UI shells reside in modules. A lightweight build (or manual copy) assembles those pieces into deployable static pages.

Asset pipeline

Shared CSS and JavaScript live in assets. Module-specific styles layer on top of the base skin. The structure keeps the repository predictable and mirrors the separation between content and presentation.

Linking patterns

Articles use standard anchor links and section IDs so that the wiki script can auto-generate table-of-contents navigation. Cross-module links (for example, pointing to the blog) use relative URLs and slug identifiers defined in metadata files.
