Publishing checklist

Scope: Release management Â· Status: Draft

Prepare content

  • Update or add documents in content/wiki or content/blog.
  • Ensure section headings use semantic <h2> elements for table-of-contents generation.
  • Cross-link related articles and posts using relative URLs.

Verify navigation

  • Run the search index script to refresh data/search/index.json.
  • Spot-check wiki and blog modules to confirm new items appear in listings.
  • Test anchor links and breadcrumbs.

Publish

  • Commit content and data updates.
  • Deploy static assets to GitHub Pages.
  • Announce major changes through a blog post when appropriate.
